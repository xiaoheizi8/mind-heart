# 阿里云百炼知识库接入指南

## 一、概述

本文档说明如何将心理健康知识库接入阿里云百炼知识库服务，实现RAG（检索增强生成）功能。

## 二、阿里云百炼知识库配置

### 2.1 创建知识库

1. 登录阿里云百炼控制台 (https://bailian.aliyun.com/)
2. 进入「知识库」页面
3. 创建知识库，配置如下：
   - **知识库名称**：mind-realm-knowledge
   - **知识库描述**：心域心理健康知识库
   - **切片方式**：建议使用「自动分段」或「语义分段」
   - **向量化模型**：text-embedding-v3

### 2.2 上传文档

将 `knowledge-base.md` 文档上传到知识库：
- 系统会自动进行文档分片
- 自动生成向量存储
- 完成后记录「知识库ID」

### 2.3 获取配置信息

获取以下配置信息：
- **API Key**：阿里云百炼API密钥
- **知识库ID**：创建的知识库ID
- **向量数据库端点**（如使用专用向量库）

## 三、应用配置

### 3.1 环境变量配置

在 `application-dev.yml` 中配置：

```yaml
# 阿里云百炼配置
spring-ai-alibaba:
  dashscope:
    api-key: your-api-key-here

# 向量存储配置
vector:
  store:
    type: alibaba  # 切换到alibaba模式
  alibaba:
    knowledge-base:
      enabled: true
      id: your-knowledge-base-id
```

或通过环境变量：
```bash
export DASHSCOPE_API_KEY=sk-xxxxxxxx
export DASHSCOPE_KB_ID=kb-xxxxxxxx
```

### 3.2 支持的配置模式

| 配置模式 | 说明 |
|---------|------|
| `vector.store.type: memory` | 本地内存模式（默认，无需配置API） |
| `vector.store.type: alibaba` | 阿里云百炼模式（需要API Key） |

### 4.2 自动分片说明

- 文档使用Markdown标题层级结构
- 阿里云百炼会自动根据段落和标题进行语义分片
- 每个切片约500-1000字
- 系统自动生成向量

### 4.3 分片优化建议

如果需要手动优化分片，可在文档中添加分隔符：
```markdown
---
SID:001
---
## 内容片段1
---
SID:002
---
## 内容片段2
```

## 五、API调用

### 5.1 问答接口

```
POST /api/v1/knowledge/qa

请求体：
{
  "question": "如何缓解考试焦虑？"
}

响应：
{
  "code": 200,
  "data": {
    "answer": "考试焦虑是常见现象，缓解方法包括：..."
  }
}
```

### 5.2 获取知识库信息

```
GET /api/v1/knowledge/info
```

## 六、注意事项

1. **API Key安全**：不要将API Key提交到代码仓库
2. **调用限制**：注意阿里云百炼的API调用配额
3. **费用**：阿里云百炼按调用量计费，注意监控使用情况
4. **内容审核**：知识库内容应符合相关法规要求

## 七、故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 返回空结果 | 未配置API Key或知识库ID | 配置正确的环境变量 |
| 调用超时 | 网络问题或API限制 | 检查网络，增加超时时间 |
| 分片效果差 | 文档结构不清晰 | 优化文档结构，添加明确标题 |

### 7.2 日志查看

查看RAG服务日志：
```bash
tail -f logs/mind-realm.log | grep AlibabaRagService
```

## 八、升级指南

### 从本地模式升级到阿里云模式

1. 在阿里云百炼创建知识库
2. 上传知识文档
3. 获取API Key和知识库ID
4. 修改配置文件，切换到alibaba模式
5. 重启应用

---

**注意**：本系统已内置本地RAG模式，即使不配置阿里云百炼也能正常提供心理健康咨询服务。
