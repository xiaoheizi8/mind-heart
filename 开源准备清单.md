## ⚠️ 需要手动处理

### 1. 更新 README 中的 GitHub 链接

**文件**: `README.md`  
**操作**: 将 `your-username` 替换为你的 GitHub 用户名

```bash
# 替换所有出现的地方
sed -i 's/your-username/YOUR_GITHUB_USERNAME/g' README.md
```

### 2. 检查敏感文件

**必须确保以下文件不在 Git 中**:

- ❌ `mind-realm-api/src/main/resources/application-dev.yml`
- ❌ `mind-realm-api/src/main/resources/application-prod.yml`
- ❌ 任何包含 API Key、密码的配置文件
- ❌ `database/` 目录下的 SQL 文件
- ❌ `docs/` 目录下的文档

**验证命令**:

```bash
# 检查是否有敏感文件被跟踪
git ls-files | grep -E "(application-dev|application-prod|\.yml|database/|docs/)"

# 如果有，使用以下命令移除
git rm --cached mind-realm-api/src/main/resources/application-dev.yml
git rm -r --cached database/
git rm -r --cached docs/
```

### 3. 清理 Git 历史 (可选)

如果之前已经提交了敏感文件，需要清理 Git 历史:

```bash
# 方法 1: 使用 git filter-branch (较慢)
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch mind-realm-api/src/main/resources/application-dev.yml' \
  --prune-empty --tag-name-filter cat -- --all

# 方法 2: 使用 BFG Repo-Cleaner (推荐,更快)
# 下载: https://rtyley.github.io/bfg-repo-cleaner/
java -jar bfg.jar --delete-files application-dev.yml
java -jar bfg.jar --delete-folders database
java -jar bfg.jar --delete-folders docs
```

```bash

# 贡献指南

感谢你对 MindRealm 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 提交 Pull Request

## 开发规范

- 遵循阿里巴巴 Java 开发规范
```

### 6. 最终检查清单

**提交前验证**:

```bash
# 1. 编译项目
mvn clean compile -DskipTests

# 2. 运行测试 (可选)
mvn test

# 3. 检查敏感文件
git status
git diff --cached

# 4. 确认 .gitignore 生效
git check-ignore -v mind-realm-api/src/main/resources/application-dev.yml
# 应该输出 .gitignore 规则

# 5. 查看将要提交的文件
git ls-files --others --ignored --exclude-standard
```

## 🚀 发布到 GitHub

```bash
# 1. 初始化 Git 仓库 (如果还没有)
git init

# 2. 添加远程仓库
git remote add origin https://github.com/YOUR_USERNAME/mind-realm.git

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "feat: 初始版本发布"

# 5. 推送
git push -u origin main

# 6. 打标签 (可选)
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```
